## Vetted findings not yet planned

Confirmed during the audit, available for promotion to plans on request:

- ~~Numeric metadata-filter divergence~~ -- promoted to plan 008, fixed in
  PR #84 (the conformance subtest now enforces parity on both backends).
- **Numeric metadata-filter comparisons diverge between backends** (bug, M,
  HIGH confidence -- discovered 2026-06-12 by the plan 006 conformance suite
  running against real Postgres). A filter like `{chapter <= 3}` with a
  numeric value works on SQLite (`json_extract` compares numerically) but
  errors on pgstore: both the pre-003 `metadata->>'key'` and the current
  `jsonb_extract_path_text` extraction are text-typed, so a non-string value
  cannot bind against the comparison (`pgx: unable to encode 3 into text
  format for text (OID 25)`). Pre-existing -- not a plan 003 regression.
  Candidate fix: cast based on the Go type of the filter value (e.g.
  `(jsonb_extract_path_text(...))::numeric` when the value is a number), with
  conformance subtest `NumericMetadataComparisonDivergence` flipping from
  skip to enforce. The conformance suite documents the gap until then.
- **Concurrency-safe migrations + uniform per-package test-DB helper** (dx/tech-debt,
  S-M, HIGH confidence -- surfaced during 012b/#89 review). Two related test-robustness
  items, neither a product bug (production migrates once per process):
  (1) the session migration's `ADD CONSTRAINT` is guarded by `EXCEPTION WHEN
  duplicate_object` (42710) but a *concurrent* UNIQUE-constraint creation collides on
  the underlying index name and raises `duplicate_table` (42P07), which the guard misses;
  the pgvector `CREATE EXTENSION IF NOT EXISTS` similarly races to 23505. Catching those
  codes makes the migrations genuinely idempotent under concurrency.
  (2) #89 isolated the httpapi and memstored pg tests into private ephemeral databases
  (`CREATE DATABASE` per test) to stop them racing the shared `MEMSTORE_TEST_PG` DB under
  parallel `go test ./...`; pgstore's own tests still share that DB (safe today because
  they run serially in-binary). A single shared `pgtest.UniqueDB(t)` helper adopted by all
  pg-using test packages would make the pattern uniform and the suite robust against any
  future parallel pg package. Both are cheap; neither blocks anything.
- **Zero test files in `cmd/memstore-mcp`** (tests, M, HIGH confidence).
  `go test ./...` reports `[no test files]` for the highest-churn entry
  point (26 commits/90 days). `cmd/memstore` and `cmd/memstored` both have
  tests to model on.
- **SQLite vector search is a full scan + full sort**
  (`search.go:183-259`, perf, M, HIGH confidence on the code, LOW urgency).
  Every vector query loads all embeddings in the namespace, computes cosine
  per row, and sorts everything before truncating to the limit. Inherent to
  SQLite-without-a-vector-index; a top-k heap would shave the sort but not
  the scan. Revisit if local fact counts reach ~50K+.
- **golangci-lint enables only 4 linters** (`.golangci.yml`: govet,
  staticcheck, unused, ineffassign; dx, S). Adding errorlint and misspell
  is cheap; a maintainer-taste call.

## Findings considered and rejected

So nobody re-audits these:

- **Export drops embeddings**: by design and documented at
  `transfer.go:21-23` ("deliberately excluded ... Re-embed after import").
  The V11 quarantine columns (`embed_failed_at`/`embed_error`) are also not
  exported; consequence is one extra failed embed attempt per poison fact
  after an import, then re-quarantine. Not worth a plan on its own; fold
  into transfer work if Export/Import is ever touched.
- **Config file perms vs umask**: rejected -- umask can only clear
  permission bits, so the 0600 in `setup_cmd.go` cannot become
  world-readable via a loose umask. The reported finding was wrong.
- **Missing `IdleTimeout` on memstored's http.Server**: rejected -- Go
  falls back to `ReadTimeout` (30s, set at `cmd/memstored/main.go:227`)
  when `IdleTimeout` is zero. Slowloris exposure is already bounded.
- **Backfill goroutine outlives shutdown** (`cmd/memstored/main.go:200-214`):
  mostly rejected -- the backfill context derives from the main ctx, so
  SIGTERM cancels it; the residual race against process exit is harmless.
- **Per-keyword FTS queries in recall** (`httpapi/recall.go:166-190`):
  rejected as a micro-optimization -- each keyword query is an indexed FTS5
  lookup over a handful of keywords, and the merge logic needs per-keyword
  hit attribution that a single OR query would lose.
- **Design docs describe unshipped features**: rejected -- the tier docs
  carry explicit status headers (e.g. `docs/tier1-graph-basics.md:3`
  "Status: draft, awaiting decisions"), and the README does not list any
  unshipped tools. The audit subagent's claim did not survive verification.
- **Unauthenticated endpoints in httpapi**: none found --
  `Handler.ServeHTTP` (`httpapi/handler.go:115-153`) enforces auth on every
  route except the deliberate `/v1/health` exemption, and memstored always
  wires a token verifier.
